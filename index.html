The Web Design Process
Typography - Typeface

Decide on the main typefaces

2-3 type faces maximum

(more typefaces = worse performance)

“Typeface” = “font-family” in CSS

The Web Design Process
Typography - Typeface

Decide on a font-stack - the list of fonts you would like to see, in order of preference, e.g.:

Raleway, Helvetica Neue, Helvetica, Lucida Sans, Arial, sans-serif

This means “use Raleway, but if we don't have that font then use Helvetica Neue, but if we don't have that font then use Helvetica...” etc.

The Web Design Process
Typography - Typeface

Provide the typefaces for non-standard fonts

Font-providers like Google fonts, typekit, and fonts.com* provide links for loading fonts

*other providers are available

“Non-standard” = fonts that aren't already installed on users' devices. Note that different platforms (Windows, Mac OS, iOS, Android, Linux, etc.) may have different fonts installed by default.

The Web Design Process
Typography - Typeface

Linked fonts quick and easy to implement (happy developers) and fast to load (happy users, happy Google)

If you can't find a link to the font, then provide files

The Web Design Process
Typography - Typeface

Font files should be in WOFF format. Not OTF. Not TTF. WOFF

If you don't have the files in WOFF format, then provide a link

If you don't have a link or WOFF files then choose a different typeface!

The Web Design Process
Typography - Other Font Sizes

Font sizes may be specified in:

  • px – for the base font size
  • rem – relative to the base font size
  • em – relative to the font size of the parent element
  • % – relative to the font size of the parent element

The Web Design Process
Typography - Other Font Sizes

Apart from the base font size, you should never use px

Why fixed sizes are bad

The Web Design Process
Typography - Other Font Sizes

A good strategy:
  • base font size in  px
  • each component has a rem font size
  • elements inside each component may have an em font size

That means everything can scale easily by changing the base font size, and all sizes remain in proportion

The Web Design Process
Typography - Other Font Sizes

All sizes, especially margins and padding, should be specified in relative units (em)

Relative = Scalable

Why fixed sizes are bad

The Web Design Process
Images

Image formats:

  • WEBP - like PNG/JPG but progressively loads(it shows up before it's finished downloading and has better compression - best for performance)
  • JPG - compressed raster images
  • PNG - compressed raster images with transparency
  • GIF - limited colour images, supports animation and progressive loading
  • SVG - scalable vector images (solid blocks, shapes, lines)

The Web Design Process

WEBP is best for raster images

SVG is best for vector images
