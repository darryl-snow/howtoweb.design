Code Constraints

Document flow
  • Everything 'flows' ↴ (L > R, T > B)

  • Consider when layout adapts between small & large screens…
  • Use a grid system!

Web Design Challenges

CSS (styling / appearance)

  • Can do a lot, but not everything

Web Design Challenges
Code Constraints

  • Default styling is different across browsers
  • Custom styling is difficult
  • Custom styling may break accessibility!
  • Design must be approached carefully, and must be tested!

Web Design Challenges
Performance

i.e. how fast everything loads

Google cares about performance

Users care about performance

Performance begins with design

Web Design Challenges
Performance

Fewer images, fonts, & less code ⇨ better performance

Web Design Challenges
Performance

Design should be done iteratively and in collaboration with a developer so that the performance can be  tested and inform ongoing design decisions.

  • Iteration
  • Collaboration

Web Design Challenges
Typography

  • Type renders differently on the web compared to other software
  • Text renders differently on different platforms
  • Fonts impact performance (especially ones with lots of glyphs, e.g. Chinese)

Web Design Challenges
Typography

  • Some fonts may not be possible
  • CSS vocabulary is different from traditional typography, creating communication issues
    • kerning = letter spacing
    • leading = line-height
    • etc.

Web Design Challenges
Device Resolution

We need to consider pixel density of different devices. Images need to be prepared for different screens (e.g. retina).

Who Are We Designing For?

Performance
  • We need to design something that can be fast!

Who Are We Designing For?
Users

Accessibility
  • We need to design something that can be accessed in as many ways as possible: mouse, keyboard, screen reader, voice control, etc.

Who Are We Designing For?
Users

Copy
  • Be clear and concise
  • Have a goal in mind before writing
  • Structure content in a logical flow
  • Use typography to build a visual hierarchy

Who Are We Designing For?
Users

Microcopy

e.g. error messages, tooltips, notifications, etc.

Microcopy can be applied to improve the user experience.

Who Are We Designing For?
Machines

Machines (e.g. search engine robots) also access the end product. For the web project to be machine-readable, the content should have a clear hierarchy and be contained in semantic markup

Who Are We Designing For?
Machines

SEO

Machines like fresh, original content, so try to avoid repeat content at all costs. They also like good performance and usability.

Who Are We Designing For?
Machines

Rich Snippets & Meta Data

Consider also that there are certain types of content that can be flagged for search engines and other services - try to include such content as much as possible

The Web Design Process

Provide a colour palette with colours specified in Hex (e.g. #FFFFFF) or RGB (e.g. 255,255,255)

The Web Design Process
Specifying Colours

Remember DRY?

If one colour is simply a variation of another, specify the transformation, not a separate colour

CSS and CSS-preprocessors can handle colour transformations!

The Web Design Process
Specifying Colours

  • lighten(colour, %)
  • darken(colour, %)
  • alpha(colour, %) (transparency/opacity)
  • hue(colour, %)
  • saturation(colour, %)
  • lightness(colour, %)
